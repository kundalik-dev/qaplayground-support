## What is QA Playground?

**QA Playground** is a modern, free web platform built specifically for QA automation engineers who want to practice browser automation without fighting production apps that weren't designed for testing.

Every element on the site has stable `id`, `data-testid`, and `data-action` attributes — so you can focus on learning Selenium, Playwright, and Cypress instead of hunting for flaky selectors.

It features four primary areas:

1. **Practice Elements** - 22 interactive UI components (inputs, buttons, tables, calendars, drag-and-drop, alerts, iframes, shadow DOM, and more) built to be automated.
2. **Bank Demo App** - A full simulated banking application (login, accounts, dashboard, transactions) for practicing end-to-end automation scenarios.
3. **Study Tracker** - Track your QA learning progress across syllabi, daily logs, and saved resources.
4. **QA Tools** - Free online utilities for daily QA workflows (JSON converter, formatter, and more coming soon).

## Guidelines

Before submitting an issue:

- **Search first** - Check if your issue or idea already exists in our active issues or discussions list.
- **Be specific** - Provide precise steps to reproduce, browser details, automation framework version, and screenshots where possible.
- **One issue per report** - Keep bug reports and feature requests focused on a single topic.
- **No sensitive data** - Ensure any credentials, tokens, or private information are removed from logs and screenshots.

_This is a support repository for bug reports and feature requests. The frontend source code is private._
